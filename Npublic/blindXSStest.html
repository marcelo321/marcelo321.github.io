After working for a company that had a bug bounty program in hackerone I realized many companies import the report's information either automatically or manually to jira instances and similar software that allows to track external reports and security vulnerabilities in general.

This can usually lead to other vulnerabilities such as Blind XSS. But the problem is that it can be really hard to test this without actually commenting in a h1 report and generating a notification to the internal team. Please I'm sorry for generating this notification to your team (if you are a program's member) but this is the only way I can think of actually testing this vulnerability.

The ideal way to test this would be to make a new report with a blind xss payload, but that's something I would get annoyed as a program member, for that reason I did it in an existing report. Sorry for the bothering!
